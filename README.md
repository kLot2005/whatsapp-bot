# ⚖️ WhatsApp Legal Bot — Бот юридической фирмы

Node.js бот на базе WhatsApp Cloud API с интеграцией в Bitrix24 CRM.

---

## 📁 Структура проекта

```
whatsapp-legal-bot/
├── config/
│   └── index.js                  # Конфигурация из .env
├── src/
│   ├── app.js                    # Точка входа, Express + Webhook
│   ├── fsm/
│   │   ├── states.js             # Константы состояний FSM
│   │   └── sessionManager.js     # CRUD сессий в Redis
│   ├── integrations/
│   │   ├── whatsapp.js           # Отправка сообщений (text, buttons, list)
│   │   └── bitrix24.js           # Создание лида в Bitrix24
│   ├── middleware/
│   │   └── messageHandler.js     # Ядро FSM — обработка состояний
│   └── utils/
│       └── validators.js         # Валидация ИИН, суммы долга
├── api-examples.json             # Примеры JSON для WhatsApp API
├── .env.example                  # Шаблон переменных окружения
└── package.json
```

## 🔄 Схема FSM (Finite State Machine)

```
┌─────────────┐
│     NEW     │ ← Нет сессии в Redis
└──────┬──────┘
       │ Любое сообщение
       ▼
┌──────────────────┐
│  AWAITING_START  │ ← Отправляем приветствие + кнопку
└──────┬───────────┘
       │ Кнопка: "Начать консультацию"
       ▼
┌────────────────────┐
│  AWAITING_CONSENT  │ ← GDPR: кнопки "Согласен" / "Отказ"
└──────┬─────────────┘
       │ Кнопка: "Согласен"
       ▼
┌─────────────────┐
│  AWAITING_NAME  │ ← Свободный текст (ФИО)
└──────┬──────────┘
       │ Любой текст
       ▼
┌────────────────┐
│  AWAITING_IIN  │ ← Валидация: ровно 12 цифр
└──────┬─────────┘    При ошибке — переспрос
       │ Валидный ИИН
       ▼
┌──────────────────────┐
│  AWAITING_PROPERTY   │ ← Кнопки: "Есть имущество" / "Нет"
└──────┬───────────────┘
       │ Нажата кнопка
       ▼
┌─────────────────┐
│  AWAITING_DEBT  │ ← Число (сумма долга)
└──────┬──────────┘    При ошибке — переспрос
       │ Валидная сумма
       ▼
┌────────────────────┐
│  AWAITING_PROBLEM  │ ← Свободный текст
└──────┬─────────────┘
       │ Любой текст
       ▼
┌───────────────────────────────────┐
│  COMPLETED  → createLead(Bitrix24)│ ← Создаём лид, отправляем подтверждение
└───────────────────────────────────┘
```

## 🗄️ Схема хранения сессии в Redis

**Ключ:** `session:{phone}`  
**TTL:** 86400 секунд (24 часа) — автоматический таймаут

```json
{
  "phone": "77001234567",
  "state": "AWAITING_IIN",
  "data": {
    "name": "Иванов Иван Иванович",
    "iin": "880101300123",
    "property": "Есть недвижимость/транспортное средство",
    "debt": "5000000",
    "problem": "Описание ситуации"
  },
  "createdAt": "2024-01-15T10:00:00.000Z",
  "updatedAt": "2024-01-15T10:05:32.000Z"
}
```

**Поведение таймаута:**  
- При каждом `saveSession()` TTL обновляется до 24 часов  
- Если пользователь молчит 24 часа — Redis автоматически удаляет ключ  
- Следующее сообщение от пользователя создаст новую сессию (приветствие)

## ⚙️ Установка и запуск

### 1. Установить зависимости
```bash
npm install
```

### 2. Настроить переменные окружения
```bash
cp .env.example .env
# Заполнить все поля в .env
```

### 3. Запустить Redis
```bash
# Локально через Docker:
docker run -d -p 6379:6379 redis:alpine
```

### 4. Запустить бота
```bash
npm start          # Продакшн
npm run dev        # Разработка (nodemon)
```

### 5. Настроить Webhook в Meta Developer Console
- URL: `https://your-domain.com/webhook`
- Verify Token: значение из `WHATSAPP_VERIFY_TOKEN`
- Подписаться на события: `messages`

## 🔌 WhatsApp Cloud API — настройка

1. Создать приложение на [developers.facebook.com](https://developers.facebook.com)
2. Добавить продукт **WhatsApp Business**
3. Получить `Phone Number ID` и `Permanent Access Token`
4. Настроить вебхук с URL вашего сервера

---

## 📦 Bitrix24 — настройка

1. Перейти: CRM → Настройки → REST API → Входящий вебхук
2. Создать вебхук с правами: `crm` (чтение/запись)
3. Скопировать URL вебхука в `BITRIX24_WEBHOOK_URL`

## 🛡️ Обработка исключений

| Ситуация | Поведение |
|---|---|
| Картинка/аудио вместо текста | Вежливое предупреждение + повтор вопроса |
| Неверный ИИН | Сообщение об ошибке + повторный запрос |
| Неверный формат суммы | Сообщение об ошибке + повторный запрос |
| Молчание 24+ часов | Redis TTL удаляет сессию автоматически |
| Ошибка Bitrix24 API | Логирование в консоль, клиент получает подтверждение |
| Ошибка WhatsApp API | Логирование в консоль |
| Пользователь отказал в GDPR | Сессия удалена, вежливое прощание |
